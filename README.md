## 🧠 How I work

- **Problem-first** — every project starts from a real, felt pain (mine, my batch's, or a client's) and a written problem hypothesis.
- **0→1 execution** — I research, define an MVP, design the flow, build it, and ship it live.
- **Ship pragmatically** — vanilla web + PWAs, Firebase/BaaS, a Node backend where a secret must stay server-side, free AI (Groq) where reasoning adds value.
- **Document like a PM** — each product has a PRD with personas, success metrics, flowcharts, roadmap and honest trade-offs.
